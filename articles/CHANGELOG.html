    Changelog

    All notable changes to this project will be documented in this file.

    The format is based on Keep a Changelog, and this project adheres to Semantic Versioning.

    • Added: New features.
    • Changed: Changes in existing functionality.
    • Deprecated: Once-stable features removed in upcoming releases.
    • Removed: Deprecated features removed in this release.
    • Fixed: Any bug fixes.
    • Security: Invite users to upgrade in case of vulnerabilities.

    [Unreleased]

    Added

    • Setting the patch version in the export tool creates a version file.
    • LevelSystem now has a method, TryLoadMainMenu, that emits a signal assigned in the inspector.
    • TirLogger
      • Now accepts a context Object.
      • Color properties for the class, method, and context logs.
    • UnityRichTextColors contains static properties for each color supported by Unity's rich text interpreter.
      • This are represented by a RichTextColor readonly struct.

    Changed

    • DrawBoxGizmo now uses the center and size its collider instead of Vector3.zero and Vector3.one respectively.
    • DrawForwardGizmo line range now goes from 1.0f to 50.0f.
    • TirGizmos capsule gizmo logic has been massively refactored.
    • LevelSystem is now a MonoSingleton.

    Fixed

    • Typos in DrawColliderGizmo.
    • DrawColliderGizmo no longer throws a null ref when not given a collider.
    • MenuPageInspector no longer fuses the path name and file name when creating a MenuState with the inspector button.
    • Possible null refs in ApplicationStateMachine when not supplied with play or pause signals.
    • RouterButton now correctly removes level load listeners instead of adding more... god damn typos.
    • RouterButton now caches its button in OnValidate even if it is not null.
      • This change fixes a bad reference that occurs when copy/pasting the component.
    • Adds missing null checks to LevelData and LevelLoader.

    [v0.0.0-alpha.10.4]

    Added

    • Script templates for PropertyAttributes and PropertyDrawers.
    • ShowIf Attribute
      • Decorate a field with [ShowIf(string targetName, object targetValue)]. This field will only be drawn in the inspector if the given target equals the target value.
    • ApplicationStateMachine has additional documentation.
    • RouterButton MonoBehaviour
      • Provides a quick way to route to various common actions.
        • Load a level by calling LevelLoadSignal.Emit
        • Quit or toggle the game's paused state with an ApplicationStateMachine
          • ApplicationStateMachine.QuitGame
          • ApplicationStateMachine.TogglePaused
        • Change the active menu page with MenuStateMachine.TransitionTo(MenuState)

    Fixed

    • Experimental build tool now correctly saves its editor prefs.
